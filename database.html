Koneksi Database

Cara menghubungkan DBeaver ke PostgreSQL yang berjalan di dalam Docker Container.

‚Üê Kembali ke Halaman Utama
1
Expose Port Database
Penting: Secara default, database di Docker terkunci. Kita harus membuka "pintu" (port forwarding) terlebih dahulu.

Edit file docker-compose.yml dan tambahkan konfigurasi ports pada service db. Ini akan memetakan port internal container ke port MacBook kita.

db:
  image: postgres:15
  ports:
    - "5432:5432" <!-- Default Port -->
...

*Jika port 5432 gagal/bentrok, lihat Langkah 6 di bawah.

2
Buat Koneksi Baru di DBeaver

Buka aplikasi DBeaver, klik icon "New Database Connection" (ikon steker listrik di pojok kiri atas), lalu pilih PostgreSQL dan klik Next.

3
Input Kredensial

Masukkan detail koneksi sesuai konfigurasi Docker kita:

  • Host: localhost
  • Port: 5432 (atau 5433/5434 jika diubah)
  • Database: postgres (atau nama DB spesifik)
  • Username: odoo
  • Password: odoo
4
Test Connection & Finish

Klik tombol "Test Connection ...". Jika muncul pesan "Connected", klik Finish.

5
Verifikasi & Troubleshooting Database

Jika database hasil restore (misal: amdione-qa) tidak muncul di DBeaver, lakukan langkah berikut:

A. Cek Kebenaran via Terminal

Pastikan database benar-benar ada secara fisik dengan bertanya langsung ke Docker:

docker compose exec db psql -U odoo -l

Jika nama database muncul di list, berarti database ADA.

B. Munculkan di DBeaver

Agar DBeaver mau menampilkan semua database (bukan hanya default):

  1. Klik Kanan koneksi > Edit Connection.
  2. Pilih tab PostgreSQL.
  3. Centang "Show all databases".
  4. Disconnect & Reconnect.

6
Masalah: "Address already in use"?
Error Docker: Jika muncul error "Bind: address already in use" saat menjalankan Docker, berarti port yang kita pilih (5432 atau 5433) sedang dipakai aplikasi lain di Mac.

Solusinya: Gunakan Port Lain. Terus naikkan angkanya (misal ke 5434) sampai berhasil.

  1. Edit docker-compose.yml: ubah ports menjadi "5434:5432" (atau angka lain yang kosong).
  2. Matikan & Nyalakan ulang: 
    docker compose down && docker compose up -d
  3. Di DBeaver, jangan lupa ubah setting Port koneksi menjadi 5434.
